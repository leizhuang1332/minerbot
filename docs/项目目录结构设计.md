### 二、关键目录/文件说明

#### 1. 核心配置（config/）

- **agent_config.yaml**：定义智能体的角色、能力、协作规则（如 DeepAgents 的 `AgentConfig` 配置），示例：

    ```YAML
    
    # agent_config.yaml
    tool_agent:
      name: "工具调用智能体"
      role: "解决需要工具的任务（计算、搜索）"
      llm: "gpt-3.5-turbo"
      temperature: 0.1
      tools: ["calculator", "search"]
    multi_agent:
      name: "协作智能体组"
      members: ["tool_agent", "chat_agent"]
      coordination_mode: "round_robin"  # DeepAgents 协作模式
    ```

- **llm_config.yaml**：统一管理 LLM 配置（适配 LangChain 1.0 的 `ChatOpenAI` 等），避免硬编码。

#### 2. 智能体核心（src/agents/）

- **[base_agent.py](base_agent.py)**：定义抽象基类 `BaseAgent`，统一所有 Agent 的接口（如 `run()`、`invoke()`），示例：

    ```Python
    
    # src/agents/base_agent.py
    from abc import ABC, abstractmethod
    from langchain_core.runnables import Runnable
    
    class BaseAgent(ABC):
        def __init__(self, config):
            self.config = config
            self.llm = self._init_llm()
            self.tools = self._init_tools()
    
        @abstractmethod
        def _init_llm(self):
            """初始化LLM（LangChain 1.0）"""
            pass
    
        @abstractmethod
        def _init_tools(self):
            """初始化工具（LangChain Tools）"""
            pass
    
        @abstractmethod
        def invoke(self, input: dict) -> str:
            """核心调用方法（适配LangChain 1.0 Runnable）"""
            pass
    ```

- **[multi_agent.py](multi_agent.py)**：基于 DeepAgents 实现多智能体协作，调用 `deepagents.MultiAgent` 并集成 LangChain 链。

#### 3. 工具模块（src/tools/）

- 所有自定义工具需遵循 LangChain 1.0 的 `Tool`/`BaseTool` 规范，示例：

    ```Python
    
    # src/tools/calculator_tool.py
    from langchain_core.tools import tool
    
    @tool
    def calculator_tool(expression: str) -> str:
        """计算数学表达式（如 "1+2*3"）"""
        try:
            result = eval(expression)  # 生产环境建议用更安全的库（如 ast）
            return f"计算结果：{result}"
        except Exception as e:
            return f"计算失败：{str(e)}"
    ```

#### 4. 环境变量（.env）

- 存储敏感信息（API 密钥），通过 `src/utils/env_utils.py` 加载，示例：

    ```Plain Text
    
    # .env
    OPENAI_API_KEY=sk-xxx
    DEEPGRAM_API_KEY=xxx
    LOG_LEVEL=INFO
    ```

#### 5. 入口文件（src/[main.py](main.py)）

- 项目启动入口，示例：

    ```Python
    
    # src/main.py
    from dotenv import load_dotenv
    from src.agents.tool_agent import ToolAgent
    from src.agents.multi_agent import MultiAgent
    from src.utils.env_utils import load_config
    
    # 加载环境变量和配置
    load_dotenv()
    agent_config = load_config("config/agent_config.yaml")
    
    def main():
        # 单智能体测试
        tool_agent = ToolAgent(agent_config["tool_agent"])
        result = tool_agent.invoke({"input": "计算 100*20 + 50"})
        print("单智能体结果：", result)
    
        # 多智能体测试（DeepAgents）
        multi_agent = MultiAgent(agent_config["multi_agent"])
        multi_result = multi_agent.invoke({"input": "先计算 100*20，再解释结果"})
        print("多智能体结果：", multi_result)
    
    if __name__ == "__main__":
        main()
    ```

### 四、使用建议

1. **环境隔离**：建议用 `venv`/`conda` 创建虚拟环境，避免依赖冲突：

    ```Bash
    
    # 创建虚拟环境
    python -m venv venv
    # 激活（Windows）
    venv\Scripts\activate
    # 激活（Linux/Mac）
    source venv/bin/activate
    # 安装依赖
    pip install -r requirements.txt
    ```

2. **配置优先**：所有可变参数（如 LLM 温度、工具列表）都放在 `config/` 目录，避免硬编码；

3. **测试驱动**：开发完核心模块后，先在 `tests/` 编写测试用例，确保功能稳定；

4. **日志规范**：通过 `src/utils/log_utils.py` 统一配置日志（输出到 `logs/` 目录），方便调试。

### 总结

1. 目录结构核心遵循 **模块化+分层设计**：配置/核心逻辑/工具/测试分离，适配 LangChain 1.0 的 Runnable 架构和 DeepAgents 的多智能体协作；
